Aprende a instalar y configurar tu PigVision.

En redes donde existen políticas de seguridad (firewall), es necesario permitir ciertas conexiones para que el sistema pueda: comunicarse con la plataforma, sincronizar información y mantenerse actualizado. Si estas configuraciones no están habilitadas, el sistema puede presentar fallas de conexión o funcionamiento.

Puertos requeridos
  • 443 TCP - Necesario para la conexión a la VPN, acceso al terminal web y servicios HTTPS.
  • 123 UDP - Utilizado para la sincronización de hora mediante NTP.
  • 53 UDP - Utilizado para la resolución de nombres de dominio (DNS).

Nota: Estos puertos deben estar habilitados para tráfico saliente (y permitir tráfico entrante únicamente como respuesta a conexiones iniciadas desde la red interna).

Dominios que deben estar permitidos (whitelist)
*.balena-cloud.com *.docker.com *.docker.io *.asimetrix.co
Servidores de sincronización de tiempo (NTP – puerto 123 UDP)
0.resinio.pool.ntp.org 1.resinio.pool.ntp.org 2.resinio.pool.ntp.org 3.resinio.pool.ntp.org
DNS

Se utiliza por defecto el servidor DNS público de Google:

8.8.8.8

(Adicional a los servidores DNS que tengan configurados internamente).
